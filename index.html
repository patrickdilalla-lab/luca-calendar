📤 Google Calendar export — two modes

There are two export buttons on the Schedule page. Pick the right one for what you're doing.

1. Export full schedule (.ics) — RECOMMENDED

Emits the standard weekly schedule as recurring events (RRULE) plus an EXDATE entry for every override day. The override events are exported alongside. This makes overrides actually replace the standard event on import — no manual cleanup needed.

One-time migration steps:

  1. Delete any existing hand-made recurring "Luca with Dad/Mom/Grandparents" events from your Google Calendar (Delete → "All events"). This avoids duplicates.
  2. Click Export full schedule (.ics) on the Schedule page.
  3. Open Google Calendar Import (button below) → choose Luca as destination → upload the file.
  4. Done. From now on, when you save an override the standard event for that day can be auto-cancelled (see mode 2).

2. Per-override sync — surgical updates

After the one-time migration, when you click a day and save a new override, use the "Export override + auto-cancel standard (.ics)" button in the day modal. That file emits a STATUS:CANCELLED entry for the standard event on that specific day plus the new override event. Importing it removes just that one occurrence and adds the override — your other weeks are untouched.

3. Export deviations only — legacy

The original Export deviations (.ics) button exports overrides + travel as standalone events without touching the standard schedule. Use only if you're not ready to migrate to mode 1 (you'll have to manually delete overlapping standard occurrences in GCal).

Open Google Calendar Import ↗

Risks & assumptions: Google Calendar's behavior on ICS re-import with matching UIDs is inconsistent. The SEQUENCE field is bumped on every export to encourage updates, but if GCal duplicates an event after a re-import, delete the older copy. Multi-day standard blocks (e.g., Dad Sun 18:00 → Wed 08:30) are split into one recurring event per weekday so EXDATE for one day doesn't accidentally cancel the others — your calendar will show 4 "Luca with Dad" entries per week instead of one long one. This is intentional for clean override semantics.

Stop .ics files from auto-opening in Apple Calendar: right-click the downloaded file in Finder → Get Info → "Open with" → choose Google Chrome (or another browser) → click Change All…

Create a Google OAuth Client ID

This lets the site call your Google Calendar directly from your browser. No data leaves your machine — the token sits in this tab's memory only.

  1. Open Google Cloud Console → New Project. Name it Luca Hub, click Create. Wait for the project to provision, then make sure it's selected in the top bar.
  2. Go to APIs & Services → Library → Google Calendar API and click Enable.
  3. Go to OAuth consent screen. Pick External. Fill: App name Luca Hub, support email = your email, developer email = your email. Save. Skip Scopes. Add your Gmail address (and the co-parent's) as a Test user.
  4. Go to Credentials → Create Credentials → OAuth client ID. Application type: Web application. Name: Luca Hub Browser.
  5. Under Authorized JavaScript origins, click Add URI and add the exact origin where this site is hosted, e.g. https://your-luca-hub.netlify.app. If you also run it from a local file, you can skip — file:// origins don't work; you'll need to host it.
  6. Click Create. Copy the Client ID (looks like 1234-abc.apps.googleusercontent.com) and paste it in the field on the Settings page.
  7. Click Connect Google Calendar. Approve the consent screen (the warning "Google hasn't verified this app" is normal for personal apps with Test users — click Continue).

Why this is safe: The Client ID is public (it's not a secret). The scope requested is https://www.googleapis.com/auth/calendar.events — events read/write only. Tokens are short-lived (1 hour) and never written to localStorage.
